Pinecraft

A comprehensive guide to installing and running a Minecraft server on Raspberry Pi

Tutorial Raspberry Pi Minecraft

Overview

Pinecraft is a Raspberry Pi–hosted Minecraft server setup guide. This documentation provides a simple, reliable, and technically accurate installation process for setting up a Java Edition Minecraft server on Raspberry Pi 4 hardware, running Raspberry Pi OS Lite (64-bit). This guide is beginner-friendly while maintaining precision for those with technical experience.

Estimated Time

30–45 minutes (including downloads, OS flashing, and initial configuration)

Prerequisites

Before you begin, ensure you have all the following requirements. This guide is designed specifically for Raspberry Pi 4 and does NOT support other models.

Hardware Requirements

Raspberry Pi 4 ONLY (Minimum 4GB RAM; 8GB recommended for optimal performance)
Does NOT work on: Raspberry Pi Zero W or Raspberry Pi Zero 2 W
Stable 5V 3A power supply (official Raspberry Pi adapter recommended)
Active cooling: heatsink or fan (thermal throttling will impact server performance)

Network & Storage

Ethernet connection strongly preferred over WiFi (more stable, lower latency)
MicroSD card (32GB or larger) flashed with Raspberry Pi OS Lite (64-bit)
External storage (optional, but recommended for large worlds)
Router access for port forwarding (25565 TCP; required for remote access)
Hardware Compatibility: This guide uses the official cat5TV/pinecraft GitHub repository as the source of truth. Unsupported hardware (Pi Zero, Pi 3, or older models) will NOT work reliably.

Installation Guide

Follow these steps in order. Complete each section before proceeding to the next.

4.1 Flash Raspberry Pi OS

Start by downloading and using the official Raspberry Pi Imager to write the OS to your MicroSD card.

1. Download Raspberry Pi Imager

Visit https://www.raspberrypi.com/software/ and download the Imager for your computer (Windows, macOS, or Linux).

2. Run the Imager and select your device

Steps 
1. Click "Choose Device"
2. Select "Raspberry Pi 4"

3. Choose the operating system

Steps 
1. Click "Choose OS"
2. Navigate to "Raspberry Pi OS (Other)"
3. Select "Raspberry Pi OS Lite (64-bit)"

4. Select your storage device

Steps 
1. Click "Choose Storage"
2. Select your MicroSD card (check capacity to confirm)

5. Configure advanced settings (IMPORTANT)

Steps 
1. Click the gear icon (Advanced Options)
2. Configure the following:
   - Hostname: (name your Pi, e.g., "pinecraft")
   - Time zone: (your location)
   - Username: pi
   - Password: (create a secure password)
   - Configure WiFi: (optional; recommended to use Ethernet)
   - Enable SSH: YES (required for remote access)
   - Raspberry Pi Connect: Enable (optional)
About Raspberry Pi Connect: This service allows remote access to your Pi via the official app. It is optional but convenient for monitoring and managing your server remotely.

6. Write the OS to the card (WARNING: DATA LOSS)

⚠️ All data on the selected drive will be erased. The Imager will display a confirmation dialog stating: "All existing data on [device name] will be erased. Are you sure you want to continue?" Double-check that you have selected the correct MicroSD card.
Steps 
1. Click "Next"
2. Review the configuration
3. Click "I understand" (acknowledging data will be erased)
4. Click "Erase and Write"
5. Wait for completion (this may take 5-10 minutes)

4.2 First Boot Configuration

Once the OS is written, insert the MicroSD card into your Raspberry Pi and power it on.

1. Boot the Pi and log in

bash 
# Log in with the username and password you created in the Imager

2. Run the configuration tool

bash 
sudo raspi-config

3. Expand the filesystem

Steps 
1. Navigate to "Advanced Options"
2. Select "Expand Filesystem"
3. Click "OK"
4. Select "Finish"
5. DO NOT reboot yet

4.3 System Preparation

Update and prepare the system with required packages.

1. Update the package manager

bash 
sudo apt update

2. Upgrade all packages

bash 
sudo apt upgrade

3. Install Git

bash 
sudo apt install git

4. Reboot the system

bash 
sudo reboot
The Pi will restart. Wait approximately 1–2 minutes for it to come back online before proceeding.

4.4 Install Pinecraft

After the reboot, clone the official Pinecraft repository and run the installer.

1. Clone the Pinecraft repository

bash 
git clone https://github.com/cat5TV/pinecraft.git
cd pinecraft

2. Run the Pinecraft installer

bash 
sudo ./install

3. Walk through the installer dialogs

Steps 
1. Click "OK" through initial dialogs
2. Confirm default install path (/opt/pinecraft)
3. Select server type (e.g., Paper, Spigot, or Vanilla)
4. Choose game mode: Survival, Creative, or Hardcore
5. Enter world seed (optional; leave blank for random)
6. Accept RAM overclock warning if prompted
7. Wait for installation to complete
The installer will automatically set up the systemd service and configure the server. Installation typically takes 3–5 minutes.

4.5 Verify Server Status

After installation completes, verify that the server is running correctly.

1. Check the service status

bash 
/etc/init.d/pinecraft status

2. Interpret the results

  • Running: The server is active and ready for connections. Proceed to Testing & Connection below.
  • Not running: Reboot the Pi and check the status again.

3. Check logs if needed

bash 
tail -f /var/log/pinecraft.log
Press Ctrl+C to exit the log view. The initial server startup may take 1–2 minutes as it generates the world.

Testing & Connection

Once the server is running, you can connect from Minecraft Java Edition.

Local Connection (Same Network)

Steps 
1. Get your Pi's local IP:
   hostname -I

2. In Minecraft Java Edition:
   Multiplayer → Direct Connect
   
3. Enter the IP address shown above (e.g., 192.168.1.100)

4. Click "Join Server"

Remote Connection (External Network)

To allow players outside your network to join, configure port forwarding on your router:

Steps 
1. Log into your router's admin panel

2. Find "Port Forwarding" settings

3. Create a new rule:
   - Protocol: TCP
   - External Port: 25565
   - Internal IP: [Your Pi's local IP]
   - Internal Port: 25565

4. Get your external IP:
   curl ifconfig.me

5. Share this IP with players (e.g., 203.0.113.45)

6. Players connect via Direct Connect using your external IP
Port Forwarding Security: Exposing your Pi to the internet carries security risks. Use firewalls, keep software updated, and consider using Minecraft authentication.

Maintenance

Manage your Pinecraft server using the following commands.

Server Control

Start Server

Start the Pinecraft service:

/etc/init.d/pinecraft start

Stop Server

Safely stop the service:

/etc/init.d/pinecraft stop

Restart Server

Restart the service:

/etc/init.d/pinecraft restart

Check Status

View current status:

/etc/init.d/pinecraft status

Backup & Recovery

Create a backup of your world and server configuration:

bash 
# Stop the server first
/etc/init.d/pinecraft stop

# Create a timestamped backup
sudo tar -czf pinecraft-backup-$(date +%Y%m%d-%H%M%S).tar.gz /opt/pinecraft/

# Restart the server
/etc/init.d/pinecraft start
Always stop the server before backing up to avoid data corruption. Backups are stored in your current directory by default.

Troubleshooting

Refer to this section if you encounter issues running your Pinecraft server.

Service is not running

Check the service status:

/etc/init.d/pinecraft status

Review logs for errors:

tail -50 /var/log/pinecraft.log

Common causes:

  • Insufficient RAM (4GB minimum, 8GB recommended)
  • Java not installed or incorrect version
  • EULA not accepted
  • File permissions issue on /opt/pinecraft

Solution: Reboot the Pi and verify all prerequisites are met. If errors persist, check logs and address the specific error message.

Cannot connect locally (same network)

Verify the server is running:

/etc/init.d/pinecraft status

Get your Pi's local IP:

hostname -I

Test connectivity from another device:

ping [Pi-IP-Address]

Verify port 25565 is listening:

sudo netstat -tlnp | grep 25565

Common causes:

  • Server process crashed or never started
  • Firewall blocking port 25565
  • Incorrect IP address entered in Minecraft client

Cannot connect remotely (external network)

Verify port forwarding is configured:

  • Log into your router
  • Confirm port 25565 TCP is forwarded to your Pi's local IP
  • Check that your external IP is correct

Test port accessibility:

curl ifconfig.me:25565 -v 2>&1 | head -20

Common causes:

  • Port forwarding misconfigured in router settings
  • ISP blocking port 25565 (contact ISP for clarification)
  • Firewall on Pi or router blocking connections
  • External IP changed (ISPs often rotate IPs; use dynamic DNS if needed)

Server is slow or lagging

Check system resource usage:

top

(Press q to exit.)

Common causes and solutions:

  • High CPU/memory usage: Reduce player count or world size
  • Thermal throttling: Ensure adequate cooling (heatsink + fan)
  • Network saturation: Use Ethernet instead of WiFi
  • Disk I/O bottleneck: Consider external SSD for world storage

Performance tuning:

  • Lower server.properties settings: view-distance, simulation-distance
  • Disable unnecessary plugins or mods
  • Monitor RAM allocation in /etc/init.d/pinecraft

Server crashes unexpectedly

Check for out-of-memory errors:

dmesg | tail -20

Review server logs:

tail -100 /var/log/pinecraft.log

Common causes:

  • Insufficient RAM allocated (increase -Xmx value)
  • Java heap space exceeded
  • Plugin or mod incompatibility
  • Corrupted world data

Solution: Review the specific error in logs. If memory-related, restart with higher -Xmx allocation. If world corruption, restore from backup.
